\# 📚 Book Store SQL Data Analysis.



\## 📌 Project Overview



This project analyzes a Book Store database using PostgreSQL. It involves creating relational tables, importing CSV datasets, and solving business problems using SQL queries.



\## 📂 Project Files



\* `Books.csv` – Books dataset

\* `Customers.csv` – Customers dataset

\* `Orders.csv` – Orders dataset

\* `BookStore\_SQL\_Project.sql` – Database creation and analysis queries

\* `README.md` – Project documentation



\## 🛠 Tools Used



\* PostgreSQL

\* SQL

\* CSV Files

\## 📈 Business Problems Solved



1\. Retrieve all Fiction books.

2\. Find books published after 1950.

3\. List customers from Canada.

4\. Show orders placed in November 2023.

5\. Calculate total stock available.

6\. Find the most expensive books.

7\. Find customers ordering more than one quantity.

8\. Retrieve orders with amount greater than $20.

9\. List all book genres.

10\. Find books with lowest stock.

11\. Calculate total revenue generated.

12\. Find average price of Fantasy books.

13\. Identify customers placing at least two orders.

14\. Find the most frequently ordered books.

15\. Show top 3 most expensive Fantasy books.

16\. Calculate quantity sold by each author.

17\. Find cities where customers spent over $30.

18\. Identify customers who spent the most.

19\. Calculate remaining stock after sales.



\## 🔑 Key Insights



\* Generated total revenue from all orders.

\* Identified top customers based on spending.

\* Determined best-selling books.

\* Analyzed stock availability and remaining inventory.

\* Evaluated sales by author and genre.
